# 分岐型小説 シナリオ生成プロンプト（完全版）

---

## メインプロンプト

```
# 役割

あなたは「分岐型小説（インタラクティブフィクション / ゲームブック）」のシナリオライターです。
読者が選択肢を選んで物語が分岐し、複数のエンディングに到達する構成を設計してください。

---

# 制作前提（ここを埋めてから依頼する）

- **ジャンル**: [例: ホラー / 恋愛 / ミステリー / コメディ / 教育 / ブランドPR]
- **ターゲット**: [例: 20代女性 / Z世代 / ビジネスパーソン]
- **尺の目安**: 各シーン [300〜800] 字、通読 [15〜30] 分（1ルートあたり）
- **分岐の深さ**: [2〜3] 段階（選択 → 分岐 → エンディング）
- **エンディング数**: [3〜5] 種類（Good / Normal / Bad / Secret など）
- **トーン**: [例: 緊張感 / 軽快 / 感動 / ダーク]
- **舞台・設定**: [例: 深夜のオフィス / 放課後の教室]
- **主人公**: [例: 新入社員の自分 / 名探偵]
- **文体**: [例: 一人称 / 三人称 / 現在形 / 過去形]
- **配信・出版形態**: [例: ゲームブック / Web小説 / 電子書籍 / note連載 / インタラクティブ小説アプリ]
- **目的**: [例: エンタメ / 商品訴求 / 教育 / 認知拡大]

---

# 出力形式（必ずこの構造で）

## 1. 作品概要（100字以内）
- フック（冒頭1段落で引き込む一文）
- 読者が体験する「選択の意味」

## 2. 登場人物
| 名前 | 役割 | 一言キャラ設定 |
|------|------|----------------|

## 3. シーン一覧（ノード形式）

各シーンを以下の形式で記述：

---
**[シーンID]**: S01
**種別**: 本編 / 分岐点 / エンディング
**前シーン**: S00（ルートの場合は「START」）
**想定文字数**: XXX字
**視点**: 一人称 / 三人称 / 内面モノローグ など
**情景描写**: （場所、時間、五感、空気感を具体的に）
**本文**:
  （地の文とセリフを含む執筆可能な散文。必要なら「——」で地の文、「」でセリフ）
**文体・演出**: （改行、余白、強調、章扉の扱いなど）
**雰囲気・テンポ**: （緊張 / 静寂 / 疾走感 など、段落のリズム指示）
**選択肢**（分岐点のみ）:
  - A: 「選択肢テキスト」 → 次シーン: S02
  - B: 「選択肢テキスト」 → 次シーン: S03
**エンディング情報**（エンディングのみ）:
  - 種類: Good / Normal / Bad / Secret
  - タイトル: 「○○エンド」
---

## 4. 分岐マップ（テキスト版）

START
├─ S01 ─┬─ [A] → S02 → ...
│       └─ [B] → S03 → ...
...

## 5. 執筆メモ
- 主要ロケーション一覧（舞台設定の整理用）
- 再利用できる共通描写・共通場面（執筆効率化用）
- 選択肢の表示形式の注意点（ゲームブック番号 / Web UI / 章末分岐 など）

---

# 執筆ルール

1. **冒頭1段落でフック**: 問いかけ・異変・衝撃の一文から始める
2. **選択肢は2〜3個**: 短く（15字以内）、直感的に、結果の方向が想像できる表現
3. **各選択に意味を持たせる**: 正解/不正解ではなく「価値観・性格・リスク許容度」の違いとして設計
4. **分岐は最大3段階**: 読者が迷子にならない深さに抑える
5. **全ルートを通しても飽きない**: 共通部分は短く、分岐後は固有の展開を必ず入れる
6. **エンディングは感情で差別化**: 同じ結末の言い回しを避け、Secretエンドを1つ入れる
7. **短編小説向け**: 1シーン1メッセージ。説明過多にしない
8. **インタラクション前提**: 「あなたならどうする？」「——さあ、選べ」など読者への呼びかけを分岐点に入れる
9. **描写は執筆可能な具体度**: 抽象的な演出指示より、実際に書ける情景・動作・内面を書く

---

# 避けること

- 4段階以上の深い分岐
- 選択肢が5個以上
- 分岐後にまたすぐ分岐（選択疲れ）
- エンディングの差がセリフ1行だけ
- 読者の選択が結果に反映されない「見せかけ分岐」
- 地の文とセリフの比率が極端（会話劇だけ / 説明だけ）

---

上記の前提に従い、完全なシナリオを出力してください。
最初に作品概要と分岐マップを示し、その後すべてのシーンをシーンID順に記述してください。
```

## 追記プロンプト①：初稿ブラッシュアップ（総合改善）

初稿が出たあと、以下をそのまま追記して送る。

```
上記シナリオを以下の観点で改善してください：

1. 各選択肢の文言を15字以内に短縮し、選びやすい表現にする
2. 最初のシーン（S01）のフックを強化し、冒頭1段落で「続きが気になる」構成にする
3. Secretエンドの到達条件を1つ追加し、隠し選択肢または隠し分岐ルートを設計する
4. 執筆効率のため、共通描写・共通場面を3つ以上特定し、執筆メモに追記する
5. 各エンディングに「読後感を残す一文」（引用・感想向けのキャッチコピー）を追加する
6. 分岐マップを更新し、変更点を箇条書きで説明する

改善後は、変更したシーンのみ全文を再出力し、それ以外は「変更なし」と明記してください。
```

## 追記プロンプト②：分岐構造の検証・修正

```
上記シナリオの分岐構造を検証し、問題があれば修正してください。

【検証項目】
1. すべてのシーンIDが分岐マップと一致しているか
2. 行き止まり（到達不能シーン）や孤立シーンがないか
3. すべてのエンディングに必ず到達できるルートがあるか
4. 分岐の深さが3段階を超えていないか
5. 連続する分岐点（選択疲れ）がないか
6. 各ルートの通読時間・文字数が指定尺（[15〜30] 分 / [XXXX] 字）以内か

【出力形式】
- 検出した問題点の一覧
- 修正方針
- 修正後の分岐マップ（全文）
- 変更・追加・削除したシーン（該当分のみ全文）
```

## 追記プロンプト③：選択肢・分岐の質を高める

```
上記シナリオの「選択の設計」を改善してください。

【改善基準】
1. 各選択肢が「価値観・性格・リスク許容度」の違いとして機能しているか
2. 「正解/不正解」ではなく、どちらを選んでも物語として成立するか
3. 選択肢の文言から、結果の方向性が想像できるか（完全なネタバレは避ける）
4. 分岐点ごとに読者への呼びかけ（「あなたなら？」「どうする？」）があるか
5. 前期の選択が後半の展開・セリフ・エンディングに反映されているか

【出力】
- 各分岐点の改善前→改善後の選択肢比較表
- 選択結果が物語に反映されるよう修正したシーン（該当分のみ全文）
- 「見せかけ分岐」があれば、結果が変わるよう修正した箇所の説明
```

## 追記プロンプト④：小説向けに尺・テンポを最適化

```
上記シナリオを小説の尺・テンポに合わせて最適化してください。

【条件】
- 各シーン: [300〜800] 字以内
- 1ルート通読: [15〜30] 分以内（目安 [3000〜6000] 字）
- 1シーン = 1メッセージ（情報を詰め込みすぎない）
- セリフは1シーンあたり必要最小限。地の文で動きと内面を補う

【作業内容】
1. 各シーンの文字数見積もりを記載
2. 長すぎるシーンは分割または削減
3. 説明過多な地の文・セリフを削る
4. テンポが落ちる箇所に短い段落・余韻・改ページを追加
5. ルートごとの通読文字数を計算し、超過ルートがあれば短縮案を提示

【出力】
- ルート別の文字数・通読時間一覧
- 尺超過があった場合の短縮方針
- 最適化した全シーン（または変更シーンのみ全文）
```

## 追記プロンプト⑤：描写・文体の具体化

```
上記シナリオの描写と文体を、実際に執筆可能なレベルまで具体化してください。

【各シーンに追加・改善する項目】
- 視点: 一人称 / 三人称 / 内面 / 焦点の移動
- 五感: 見え方・音・匂い・触覚・温度
- 動作: 誰が・何を・どの順序で
- 時間帯・照明: 朝 / 夕 / 夜、明るさの変化
- 小道具・服装: 物語に効く詳細のみ

【執筆効率化】
1. 同じロケーション・同じ時間帯で書けるシーンをグループ化
2. 共通描写（全ルートで使い回せる情景・内面）をリストアップ
3. 執筆ボリュームの目安（短編 / 中編 / 全ルート合計字数）を執筆メモに追記

【出力】
- 更新した全シーン（情景描写・本文欄を重点的に）
- 執筆順序の提案（ロケーション別）
- 共通描写一覧と再利用箇所
```

## 追記プロンプト⑥：エンディングの差別化・リプレイ性

```
上記シナリオのエンディング設計を強化してください。

【改善基準】
1. 各エンディングの「感情」が明確に異なる（安堵/後悔/驚き/感動/恐怖 など）
2. エンディングタイトルが印象に残る（6字以内推奨）
3. Good / Normal / Bad / Secret の差がセリフ1行だけで終わっていない
4. Secretエンドは「気づきにくいが理不尽ではない」条件で到達できる
5. 各エンディング末尾に「もう一度読み直す」への誘導文を入れる
6. 各エンディングに引用・感想向けキャッチコピー（20字以内）を付ける

【出力】
- エンディング比較表（種類 / タイトル / 感情 / 到達ルート / キャッチコピー）
- 改善したエンディングシーン（全文）
- Secretエンドの到達条件と、ヒントをどこに仕込むかの説明
```

## 追記プロンプト⑦：出版・配信形態別最適化

```
上記シナリオを [ゲームブック / Web小説 / 電子書籍 / note連載 / インタラクティブ小説アプリ名] 向けに最適化してください。

【形態別の調整】
- **ゲームブック**: 選択肢を番号付きで記載（「→ 42ページへ」形式）
- **Web小説 / note**: 章末分岐、リンクまたは番号選択の明示
- **アプリ**: 選択肢UIの表示タイミング（段落区切り / 章末）
- **冒頭のフック**: 最初の100字以内で離脱されない工夫
- **再読・全ルート収集**: エンディング一覧や分岐回想を促す構成
- 形態の文字数上限・更新頻度に合わせた調整

【出力】
- 形態向けの変更点一覧
- 選択肢の表示形式の注意（番号 / リンク / 章扉 など）
- 最適化後のS01（冒頭）と全分岐点シーン（全文）
```

## 推奨ワークフロー

| 順番 | 使うプロンプト | 目的 |
|------|----------------|------|
| 1 | メインプロンプト | 初稿生成 |
| 2 | 追記① 総合改善 | 全体の質を底上げ |
| 3 | 追記② 分岐検証 | 構造の整合性確認 |
| 4 | 追記③ 選択肢改善 | インタラクションの核を強化 |
| 5 | 追記④ 尺最適化 | 小説向けテンポ調整 |
| 6 | 追記⑤ 描写具体化 | 執筆可能な散文に |
| 7 | 追記⑥ エンディング強化 | リプレイ性・読後感 |
| 8 | 追記⑦ or ⑧ | 出版形態 or 目的別調整 |
| 9 | 追記⑨ 最終チェック | 納品前レビュー |

すべてを毎回通す必要はありません。初稿 → ① → ② → ⑨ だけでも十分な精度が出ます。本執筆前は⑤、公開前は⑦を足すとよいです。
